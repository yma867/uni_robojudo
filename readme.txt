================================================================================
  G1 真机连接与 RoboJuDo 部署指南（个人电脑开发 + G1 机载运行）
  适用：HKUSTGZ 校园 WiFi、公用机器人、不设静态 IP
================================================================================

【总体原则】
  • 仿真 / 改代码：在个人电脑上完成
  • 真机跑程序：在 G1 机载开发机（PC2）上完成，不要在个人电脑上直接控真机
  • 公用机器人不能设静态 IP → 每次开机先在 G1 显示屏前查当前 IP，再 SSH

【关键信息备忘】
  • 开发机用户名：unitree
  • 默认密码：123（若实验室改过，以实际为准）
  • WiFi 名称：HKUSTGZ
  • 网线专用 IP（仅插网线时）：192.168.123.164  ← 日常 WiFi 模式不用这个
  • WiFi IP：每次开机可能变化，需在 G1 上查看（例如 10.4.144.167）

【路径说明】
  • 下文「项目根目录」指本仓库 clone 到本地的路径，每人不同，请自行替换
  • G1 上建议统一放在：~/uni_robojudo


================================================================================
  第一部分：第一次设置（整台 G1 / 整个实验室只需做一次）
================================================================================

说明：若 G1 已经能连 HKUSTGZ 且能登录桌面，可跳过「1.1 开 WiFi」，
      直接从「1.2 安装 RoboJuDo 环境」开始。


--------------------------------------------------------------------------------
1.1 第一次开启 WiFi（若尚未连接 HKUSTGZ）
--------------------------------------------------------------------------------

需要：G1 接显示器 + 键盘 + 鼠标，或临时插一次网线。

方法 A — 在 G1 桌面上操作（最简单）：
  1. G1 开机，登录开发机（用户 unitree）
  2. 右上角点 WiFi 图标 → 选择 HKUSTGZ → 输入密码
  3. 连接成功后，打开终端，确认 IP：
       ip a show wlan0 | grep inet
     记下 inet 后面的地址，例如：10.4.144.167

验证个人电脑能否 SSH（个人电脑也需连接 HKUSTGZ）：
  ping <刚查到的IP>
  ssh unitree@<刚查到的IP>


--------------------------------------------------------------------------------
1.2 在 G1 开发机上安装 RoboJuDo 环境（只需装一次）
--------------------------------------------------------------------------------

在 G1 终端执行（显示器前操作，或已 SSH 进去后操作均可）：

  # 若还没有代码，从个人电脑传过来（在个人电脑终端执行）：
  # scp -r <项目根目录> unitree@<G1的IP>:~/uni_robojudo

  cd ~/uni_robojudo

  # Python 环境
  conda create -n robojudo python=3.11 -y
  conda activate robojudo
  pip install torch --index-url https://download.pytorch.org/whl/cpu
  pip install -e .

  # 真机必需：Unitree C++ SDK（先按官方文档装 unitree_sdk2，再执行）
  python submodule_install.py unitree_cpp

  # 验证
  python -c "from robojudo.environment import UnitreeCppEnv"

  # 仿真相关（mujoco_viewer）在 G1 上可不装；真机不需要。


--------------------------------------------------------------------------------
1.3 第一次真机跑通（建议先在仿真验证过）
--------------------------------------------------------------------------------

在个人电脑上先跑仿真（可选但强烈建议）：
  conda activate robojudo
  cd <项目根目录>
  python scripts/run_pipeline.py -c g1_locomimic_beyondmimic

在 G1 上跑真机：
  conda activate robojudo
  cd ~/uni_robojudo
  python scripts/run_pipeline.py -c g1_locomimic_beyondmimic_real

真机遥控器操作：
  • A 键      ：急停（阻尼模式）
  • Start     ：进入 Mimic（舞蹈模式）
  • Select    ：回到 Loco（AMO 站立）
  • R1        ：下一个舞蹈
  • L1        ：上一个舞蹈
  Mimic 列表：0=LocoMode, 1=Dance_wose, 2=Violin, 3=Waltz
  典型流程：站立稳定 → Start → R1 切到 Dance_wose

跑程序前检查：
  □ 关闭 G1 官方运控程序（避免抢控制权）
  □ 机器人周围留足空间
  □ 急停 A 键可用
  □ prepare 阶段机器人放在地面上


================================================================================
  第二部分：每次开机标准流程（公用 G1，IP 可能变化）
================================================================================

每次使用 G1 前，按下面顺序做。预计耗时：开机 2 分钟 + 查 IP 1 分钟 + SSH 30 秒。


--------------------------------------------------------------------------------
步骤 1：G1 开机
--------------------------------------------------------------------------------

  1. 按 Unitree 说明开机（短按 → 长按等）
  2. 等待开发机（PC2）启动到 Ubuntu 桌面，约 1～3 分钟
  3. 确认右上角 WiFi 已连接 HKUSTGZ
     • 若未自动连接：点 WiFi → 选 HKUSTGZ → 输入密码
     • 若 WiFi 图标灰色：在终端执行
         sudo rfkill unblock all
         sudo nmcli radio wifi on
       再手动连 HKUSTGZ


--------------------------------------------------------------------------------
步骤 2：在 G1 显示屏前查看当前 IP（必做）
--------------------------------------------------------------------------------

在 G1 上打开终端（Ctrl+Alt+T 或应用程序里找 Terminal），执行：

  ip a show wlan0 | grep "inet "

输出示例：
  inet 10.4.144.167/22 brd 10.4.147.255 scope global dynamic noprefixroute wlan0

当前 IP 就是 inet 后面、斜杠前面的部分 → 本例为 10.4.144.167

备用命令：
  hostname -I
  ifconfig wlan0

把 IP 记在备忘录或实验室群里，例如：
  「今天 G1 IP：10.4.144.167」

注意：
  • 公用机不能设静态 IP，这个地址每次可能不同，必须以当天查到的为准
  • 不要用 192.168.123.164（那是插网线时的地址）


--------------------------------------------------------------------------------
步骤 3：个人电脑连接同一 WiFi
--------------------------------------------------------------------------------

  1. 个人电脑连接 HKUSTGZ（与 G1 同一网络）
  2. 打开终端，测试连通：
       ping -c 4 <步骤2查到的IP>

  • 若 ping 通 → 继续步骤 4
  • 若 ping 不通 → 可能原因：
      - 个人电脑未连接 HKUSTGZ
      - IP 抄错了
      - 校园网 AP 隔离（设备间不能互访）→ 问实验室网管，或临时用网线
      - G1 WiFi 还没连上 → 回步骤 1


--------------------------------------------------------------------------------
步骤 4：个人电脑 SSH 登录 G1
--------------------------------------------------------------------------------

  ssh unitree@<步骤2查到的IP>

  示例：
  ssh unitree@10.4.144.167

  首次连接会问是否信任指纹，输入 yes
  密码：123（或实验室实际密码）

  登录成功提示符类似：
  (base) unitree@ubuntu:~$

之后可用 Cursor、VS Code Remote SSH 等工具远程编辑 G1 上的代码。


--------------------------------------------------------------------------------
步骤 5：同步代码（若在个人电脑上改过代码）
--------------------------------------------------------------------------------

在个人电脑终端执行（把 IP 换成当天查到的）：

  scp -r <项目根目录> unitree@<IP>:~/uni_robojudo

或在 G1 上 git pull（若代码在 git 仓库里）：

  cd ~/uni_robojudo && git pull


--------------------------------------------------------------------------------
步骤 6：在 G1 上启动真机程序
--------------------------------------------------------------------------------

SSH 进去后执行：

  conda activate robojudo
  cd ~/uni_robojudo
  python scripts/run_pipeline.py -c g1_locomimic_beyondmimic_real

程序会先 prepare（平滑过渡到站立姿态）→ 再用遥控器切舞蹈。

结束 / 异常时：按遥控器 A 键急停。


--------------------------------------------------------------------------------
步骤 7：用完后（公用机器人礼仪）
--------------------------------------------------------------------------------

  □ 按 A 急停，确认进入阻尼模式
  □ 关闭 RoboJuDo 程序（Ctrl+C）
  □ 若实验室要求，恢复官方运控或关机
  □ 不要把个人 WiFi 密码、静态 IP 写死在公用机上

  第三部分：快速命令速查

【G1 上查 IP】
  ip a show wlan0 | grep "inet "

【个人电脑上连接 G1】

【个人电脑传代码到 G1】

【WiFi 连不上时（G1 上）】
  sudo rfkill unblock all
  sudo nmcli radio wifi on
  nmcli device wifi connect "HKUSTGZ" password "密码"


================================================================================
  第四部分：常见问题
================================================================================

Q: 个人电脑 ping 不通 G1，但 G1 显示已连 WiFi？
A: 1) 确认 IP 是 wlan0 的、当天的；2) 确认个人电脑也连 HKUSTGZ；
   3) 问网管是否 AP 隔离；4) 临时用网线 + 192.168.123.164。

Q: 每次开机都要重装 RoboJuDo 吗？
A: 不用。环境装一次即可，每次只需查 IP → SSH → 激活 conda → 跑程序。

Q: WiFi 每次开机都要重配吗？
A: 一般不用（autoconnect 开启时）。若 WiFi 图标不亮，执行 rfkill unblock。

Q: 能否不用 SSH，只在 G1 显示屏前操作？
A: 可以。查 IP 那步可省略，直接在 G1 终端跑程序即可；SSH 只是为了远程编辑方便。

Q: net_if 要改吗？
A: 程序跑在 G1 本机时，配置里 net_if="eth0" 一般不用改（G1 内部通信用）。
